## 一、产品概述

### 1.1 产品名称

期货问答智能体（Futures Agent）

### 1.2 产品定位

一款面向期货交易领域的 AI 问答工具，帮助用户快速获取期货知识、查询行情数据、进行技术分析，并通过可自定义的知识库实现个性化信息注入。

### 1.3 产品愿景

让每个期货交易者都拥有一个懂期货、记得住、能分析的 AI 助手。

### 1.4 核心价值主张

| 维度 | 传统方式 | 本产品的解法 |
|------|----------|-------------|
| 获取知识 | 翻手册、搜网页、问人 | 直接提问，秒级回答 |
| 查行情 | 打开交易软件切换查看 | 对话中直接查询，上下文连贯 |
| 做分析 | 手动画图、计算指标 | 输入品种名，自动生成分析报告 |
| 知识管理 | 散落各处的文档、笔记 | 统一上传到知识库，AI 自动索引 |
| 学习曲线 | 新手需要大量时间学习 | AI 根据用户水平自适应回答深度 |

## 二、目标用户

### 2.1 用户画像

#### 用户 A：期货新手（小白）

- **背景**：对期货感兴趣，刚开户或准备开户
- **痛点**：不知道什么是保证金、什么是交割月、交易规则是什么
- **核心需求**：用通俗语言解释期货概念，避免专业术语轰炸
- **使用场景**："什么是强平？强平是怎么算的？"

#### 用户 B：活跃交易员

- **背景**：有 1-3 年期货交易经验，每天看盘
- **痛点**：需要快速查看行情、技术指标，但不想在多个工具间切换
- **核心需求**：对话中直接查询行情 + 技术面分析
- **使用场景**："帮我看看螺纹钢现在什么价位，MACD 金叉了吗？"

#### 用户 C：投研/分析师

- **背景**：机构从业人员，需要快速出报告
- **痛点**：数据分散、报告撰写耗时
- **核心需求**：上传研报到知识库，AI 理解后辅助分析
- **使用场景**："我上传了一份铁矿石供需报告，帮我总结核心观点"

### 2.2 用户需求优先级

| 需求 | 小白 | 交易员 | 分析师 | 优先级 |
|------|:----:|:------:|:------:|:------:|
| 期货基础知识问答 | P0 | P2 | P3 | **P0** |
| 实时行情查询 | P2 | P0 | P1 | **P0** |
| 技术指标分析 | P3 | P0 | P0 | **P0** |
| 知识库上传管理 | P2 | P1 | P0 | **P0** |
| 用户记忆/偏好 | P1 | P0 | P1 | **P1** |
| 策略建议 | P2 | P0 | P0 | **P1** |
| 风控评估 | P3 | P1 | P0 | **P1** |
| 持仓分析 | P3 | P1 | P0 | **P2** |
| 策略回测 | P3 | P2 | P0 | **P2** |
| 新闻情绪分析 | P2 | P1 | P1 | **P2** |

### 3.2 F1 智能对话（P0）

#### 3.2.1 功能描述

用户通过自然语言与 AI 对话，系统自动识别意图并调用对应能力回答问题。

#### 3.2.2 用户故事

- 作为用户，我希望直接打字提问就能得到回答，不需要点选菜单
- 作为用户，我希望 AI 能理解"螺纹"指的是"螺纹钢"
- 作为用户，我希望回答附带数据来源，方便我验证

#### 3.2.3 交互流程

```
用户输入问题
    │
    ▼
┌──────────────┐
│  意图识别     │  识别问题属于哪个领域
└──────┬───────┘
       │
       ├── 知识类 → 检索 RAG 知识库 → 组织回答
       ├── 行情类 → 调用行情 API → 展示数据 + 简要解读
       ├── 分析类 → 拉取 K 线 → 计算指标 → 生成分析报告
       ├── 策略类 → 结合用户偏好 → 给出策略建议
       └── 通用类 → 直接对话回答
       │
       ▼
展示回答 + 来源引用 + 相关推荐问题
```

#### 3.2.4 回答格式要求

- 知识类回答：结论先行 + 详细说明 + 来源引用
- 行情类回答：关键数据（价格/涨跌幅/成交量）+ 简要解读
- 分析类回答：当前趋势判断 + 关键指标 + 支撑/压力位
- 所有建议类回答必须附带风险提示："以上信息仅供参考，不构成投资建议"

#### 3.2.5 验收标准

| 用例 | 预期结果 |
|------|----------|
| "什么是保证金？" | 准确解释保证金概念，附带来源 |
| "螺纹现在多少钱？" | 返回螺纹钢最新价格（延迟 < 3 秒） |
| "帮我分析铁矿石" | 返回技术分析摘要（趋势+关键指标） |
| "你好" | 正常打招呼，引导用户提问 |
| 连续追问 3 轮 | 保持上下文连贯 |

### 3.3 F2 知识库管理（P0）

#### 3.3.1 功能描述

用户可通过 Web 界面自主上传、管理知识库文档，AI 将自动解析并向量化这些文档，在问答时优先检索用户知识库内容。

#### 3.3.2 用户故事

- 作为交易员，我希望上传自己的交易笔记，AI 能在回答时参考我的经验
- 作为分析师，我希望上传研报，让 AI 理解特定品种的分析框架
- 作为用户，我希望管理多个知识库（如"螺纹钢专题"、"期权入门"）
- 作为用户，我希望看到文档是否已完成索引，避免上传后无法检索

#### 3.3.3 上传方式

**方式一：文件上传**

- 拖拽文件到上传区域，或点击选择文件
- 支持格式：PDF、DOCX、XLSX、CSV、MD、TXT、HTML
- 单文件上限：50 MB
- 支持批量选择多个文件

**方式二：文本粘贴**

- 提供一个富文本编辑器
- 用户可直接粘贴文本内容
- 支持 Markdown 格式
- 输入标题和正文即可保存

**方式三：URL 抓取**

- 输入一个网页 URL
- 系统自动抓取网页正文内容
- 用户预览抓取结果，确认后入库

#### 3.3.4 知识库管理页面

**列表视图**：

| 字段 | 说明 |
|------|------|
| 知识库名称 | 可编辑 |
| 文档数量 | 该知识库下的文档总数 |
| 可见性 | 私有 / 公开 |
| 创建时间 | 创建日期 |
| 状态 | 正常 / 索引中 / 异常 |

**文档视图**（进入某个知识库后）：

| 字段 | 说明 |
|------|------|
| 文档名称 | 可编辑 |
| 来源 | 上传/粘贴/URL |
| 大小 | 文件大小或字符数 |
| 索引状态 | 已完成 / 索引中 / 失败 |
| 上传时间 | 上传日期 |
| 操作 | 查看 / 编辑 / 删除 / 重索引 |

#### 3.3.5 验收标准

| 用例 | 预期结果 |
|------|----------|
| 上传 1 个 PDF 文件 | 文件上传成功，状态变为"索引中"→"已完成" |
| 粘贴 5000 字文本 | 保存成功，可被检索到 |
| 输入有效 URL | 抓取成功，显示预览，确认后入库 |
| 上传 50MB+ 文件 | 拒绝上传，提示文件过大 |
| 上传 .exe 文件 | 拒绝上传，提示格式不支持 |
| 删除文档 | 文档从索引中移除，回答不再引用 |
| 上传后提问 | 回答中引用用户知识库内容 |

### 3.4 F3 行情查询（P0）

#### 3.4.1 功能描述

查询期货品种的实时/延迟行情数据。

#### 3.4.2 覆盖范围

中国四大期货交易所的所有上市品种：

| 交易所 | 代表品种 |
|--------|----------|
| 上期所 (SHFE) | 螺纹钢、铁矿石、铜、铝、原油、黄金、白银 |
| 大商所 (DCE) | 豆粕、棕榈油、玉米、焦炭、铁矿石 |
| 郑商所 (CZCE) | 白糖、棉花、PTA、甲醇、玻璃 |
| 中金所 (CFFEX) | 沪深300、中证500、国债期货 |

#### 3.4.3 数据字段

| 字段 | 说明 |
|------|------|
| 最新价 | 当前最新成交价 |
| 涨跌幅 | 相对于昨收的涨跌百分比 |
| 涨跌额 | 相对于昨收的涨跌绝对值 |
| 今开/最高/最低 | 当日价格区间 |
| 成交量 | 累计成交量（手） |
| 持仓量 | 未平仓合约数量 |
| 昨收/结算价 | 前一日收盘/结算价 |

#### 3.4.4 验收标准

| 用例 | 预期结果 |
|------|----------|
| "螺纹现在什么价格？" | 返回最新价、涨跌幅、成交量 |
| "今天铜涨了吗？" | 返回涨跌额和涨跌幅 |
| "螺纹钢主力合约" | 识别并返回成交量最大的合约 |
| 交易所休市时查询 | 返回最后交易日数据，标注"休市中" |

### 3.5 F4 技术分析（P0）

#### 3.5.1 功能描述

根据用户指定的品种和周期，生成技术面分析报告。

#### 3.5.2 技术指标

**必选指标**（P0 必须支持）：

| 指标 | 说明 |
|------|------|
| 均线系统 | MA5/MA10/MA20/MA60 |
| MACD | 趋势指标，金叉/死叉判断 |
| RSI | 相对强弱指标，超买超卖判断 |
| 布林带 | 价格通道，突破判断 |

**可选指标**（P1 支持）：

| 指标 | 说明 |
|------|------|
| KDJ | 随机指标 |
| 成交量分析 | 量价配合 |
| 支撑/压力位 | 关键价位 |
| 趋势线 | 自动绘制 |

#### 3.5.3 分析报告模板

```
【品种】螺纹钢主力合约 (rb2510)
【周期】日线

一、趋势判断
   当前处于 [上升趋势 / 下降趋势 / 震荡]
   依据：价格在 MA20 之上/下，MACD 金叉/死叉

二、关键指标
   MA5: xxx   MA10: xxx   MA20: xxx   MA60: xxx
   MACD: DIF=xxx, DEA=xxx, 柱=xxx (金叉/死叉)
   RSI(14): xxx (超买区/中性/超卖区)
   布林带: 上轨=xxx, 中轨=xxx, 下轨=xxx

三、支撑与压力
   支撑位: xxx (MA20/前低/布林下轨)
   压力位: xxx (前高/布林上轨)

四、综合判断
   [简要总结，2-3 句话]

⚠️ 以上分析仅供参考，不构成投资建议
```

#### 3.5.4 验收标准

| 用例 | 预期结果 |
|------|----------|
| "分析一下螺纹钢日线" | 返回完整技术分析报告 |
| "铁矿 MACD 金叉了吗" | 返回 MACD 状态及是否金叉 |
| "螺纹现在超买还是超卖" | 返回 RSI 值及判断 |

### 3.6 F5 用户记忆（P1）

#### 3.6.1 功能描述

系统记住用户的偏好和历史交互，提供个性化体验。

#### 3.6.2 记忆内容

| 记忆类型 | 记住什么 | 举例 |
|----------|----------|------|
| 关注品种 | 用户经常问的品种 | "他总问螺纹和铁矿" |
| 交易风格 | 短线/中线/长线 | "偏好趋势跟踪策略" |
| 风险偏好 | 保守/中等/激进 | "止损通常设在 2% 以内" |
| 知识水平 | 新手/进阶/专业 | "不需要解释基础概念" |
| 历史对话 | 重要的历史问答摘要 | "上周问过螺纹钢支撑位" |

#### 3.6.3 用户控制

- **查看记忆**：用户可以看到系统记住了什么
- **修正记忆**：用户可以删除或修改错误的记忆
- **关闭记忆**：用户可以关闭个性化记忆功能

#### 3.6.4 验收标准

| 用例 | 预期结果 |
|------|----------|
| 连续 3 天问螺纹相关问题 | 系统记住"关注螺纹钢" |
| 下次问"现在什么价位" | 自动理解为"螺纹钢现在什么价位" |
| 用户说"我主要做短线" | 记住交易风格，后续回答倾向短线视角 |
| 用户在记忆中删除"铁矿石" | 后续不再自动关注铁矿石 |

### 3.7 F6 策略建议（P1）

#### 3.7.1 功能描述

根据品种行情和用户偏好，给出交易策略参考建议。

#### 3.7.2 策略类型

| 策略 | 适用场景 |
|------|----------|
| 趋势跟踪 | 明确上涨或下跌趋势 |
| 区间交易 | 震荡行情 |
| 突破交易 | 关键价位突破 |
| 跨期套利 | 同一品种不同合约价差异常 |

#### 3.7.3 输出格式

```
【策略】螺纹钢 - 趋势跟踪（日线级别）

入场条件：
- 方向：多/空
- 入场位：xxx（理由：突破 MA20 + MACD 金叉）

止损：xxx（依据：前低/布林下轨/固定比例）
止盈：xxx（依据：前高/趋势目标位）

仓位建议：
- 建议仓位：总资金的 x%
- 风险回报比：1:x

持有周期：预计 x-x 个交易日

⚠️ 以上仅供参考，不构成投资建议
```

### 3.8 F7 风控评估（P1）

#### 3.8.1 功能描述

用户输入当前持仓，系统评估风险并给出建议。

#### 3.8.2 输入

- 品种名称
- 持仓方向（多/空）
- 持仓数量（手数）
- 入场价格
- 总资金

#### 3.8.3 输出

| 评估项 | 说明 |
|--------|------|
| 当前盈亏 | 浮动盈亏金额和比例 |
| 风险敞口 | 持仓占资金比例 |
| 最大回撤 | 近期最大浮亏 |
| 强平价格 | 保证金不足时的强平价格 |
| VaR 值 | 在一定置信度下的最大可能亏损 |
| 建议 | 减仓/加仓/持有建议 |

## 四、非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 问答响应时间 | P50 < 3s, P95 < 8s | 从用户发送问题到首字出现 |
| 行情查询延迟 | < 2s | 行情 API 调用到返回 |
| 知识库检索 | < 1s | RAG 检索完成时间 |
| 文件上传 | < 10s（50MB 以内） | 上传完成 |
| 文档索引 | < 60s（10000 字以内） | 向量化完成 |
| 并发用户 | 支持 100 人同时在线 | MVP 阶段目标 |

### 4.2 可用性

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 99.5%（MVP）/ 99.9%（生产） |
| 数据可用性 | 交易日 100% 覆盖行情数据 |

### 4.3 安全要求

| 要求 | 说明 |
|------|------|
| 用户数据隔离 | 每个用户的知识库和记忆严格隔离 |
| 传输加密 | 全链路 HTTPS |
| 敏感信息过滤 | 回答中不泄露其他用户数据 |
| API 限流 | 单用户每分钟最多 30 次请求 |

### 4.4 合规要求

| 要求 | 说明 |
|------|------|
| 免责声明 | 所有策略/分析类回答必须附带免责声明 |
| 不构成投资建议 | 明确标注"仅供参考，不构成投资建议" |
| 数据来源标注 | 行情和分析数据标注来源 |
| 用户协议 | 用户使用前需同意用户协议 |

## 六、数据指标

### 6.1 核心指标

| 指标 | 定义 | 目标 |
|------|------|------|
| DAU | 日活跃用户数 | MVP 阶段 50+ |
| 日均对话数 | 每日对话 session 数 | MVP 阶段 200+ |
| 平均对话轮次 | 每个 session 的平均轮次 | > 5 轮 |
| 知识库上传量 | 每日新增文档数 | > 20 篇/天 |
| 回答满意度 | 用户点赞/点踩比例 | 点赞率 > 80% |

## 八、风险与约束

### 8.1 技术风险

| 风险 | 概率 | 影响 | 应对 |
|------|:----:|:----:|------|
| LLM 幻觉导致错误信息 | 高 | 高 | 知识库检索增强 + 安全校验 + 免责声明 |
| 行情数据延迟或缺失 | 中 | 中 | 多数据源备份 + 标注数据时间 |
| 向量检索不准确 | 中 | 中 | 混合检索（向量 + 关键词）+ 调优 |
| 高并发下响应慢 | 低 | 中 | CDN + 缓存 + 异步处理 |

### 8.2 业务风险

| 风险 | 概率 | 影响 | 应对 |
|------|:----:|:----:|------|
| 用户上传违规内容 | 中 | 高 | 内容审核 + 举报机制 |
| 用户依赖 AI 建议造成亏损 | 中 | 高 | 强免责声明 + 用户协议 |
| 交易所规则变更未同步 | 低 | 中 | 定期人工审核知识库 |

### 8.3 约束条件

| 约束 | 说明 |
|------|------|
| 成本 | LLM API 调用成本需控制在 $X/月以内 |
| 数据 | AkShare 免费接口有调用频率限制 |
| 合规 | 不提供具体买卖点建议，仅提供分析参考 |
